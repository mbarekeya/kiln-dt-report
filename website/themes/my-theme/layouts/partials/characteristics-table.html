Characteristics Description
C1 System under study An industrial wood drying kiln located at La Doré, Quebec, containing two rail systems for lumber loading, a reversible fan, steam heating coils, ventilation dampers, and sensors for temperature and humidity monitoring.
C2 Physical acting components dampers: Ventilation dampers with gradual aperture control (0-100% in steps of 25%). Minimum opening of 20% maintained during active drying phases to evacuate moisture. Fully closed during Heating and Equalization phases. Opened automatically by the DT when DTAB exceeds 25C; closed when DTAB falls below 5C.
door: Sliding door that closes at cycle start and opens manually by operator at cycle end
fan: Circulates hot air throughout the kiln chamber; speed varies by drying phase and direction reverses every 2 simulated hours
heater_coils: Steam-fed heating coils that raise dry-bulb temperature progressively from ambient to target Ts across drying phases
C3 Physical sensing components th_sensor: Wet-bulb temperature sensor (Th), used with Ts to compute relative humidity via psychrometric equations
ts_sensor: Dry-bulb temperature sensor (Ts), measures the air temperature inside the kiln chamber in degrees Celsius
C4 Physical-to-virtual interaction phase_data_trans: Current drying phase index and name, air velocity, fan direction, duration remaining, and progress percentage
sensor_data_trans: Published every 2 real seconds to RabbitMQ queue wood.kiln.sensor as JSON payload containing Ts, Th, derived RH, estimated MC, current phase, elapsed hours, fan speed, fan direction, damper aperture percentage, Ts adjustment, last control action, and cycle status
C5 Virtual-to-physical interaction Operator-triggered: cycle start via RabbitMQ (wood.kiln.start) and stop via HTTP POST /api/stop. Automatic DT control on wood.kiln.control: Ts setpoint adjustment (+-2C steps, +-6C cumulative cap) for SLOW_DRYING, FAST_DRYING, EXTEND_CYCLE, CHECK_OVERDRYING; damper aperture (25% steps, 20% minimum during active drying) for DTAB HIGH/LOW. All automatic actions restricted to active drying phases (not Heating/Equalization). Operator override via Reset DT Adjustments button resets Ts to 0 and dampers to minimum.

C8 Multiplicities Single kiln DT with one active drying cycle at a time. No composition of multiple DTs. One validated scenario loaded at a time. Multi-kiln multiplicity identified as a planned extension.
C9 Life-cycle stages Supports the service phase only: execute (batch validation and real-time monitoring), analyse (deviation detection and Z3 formal verification), evolve (automatic Ts and damper adjustment during active drying phases), save (sensor history and post-drying report), and terminate (cycle completion or forced stop). DT represents the kiln as-operated.

recipe_data: R40 (EPN/PG, target 16%) and R44 (SAP, target 14%) drying recipes with 7 phases
ward_clustering_model: Hierarchical Ward clustering (scikit-learn) used for scenario D batch loading optimization

Data:

C11 Tooling and enablers chartjs: Chart.js frontend rendering real-time temperature and MC curves, phase progress bars, damper aperture bar, Ts adjustment indicator, and KPI cards at port 8080
docker: Docker Compose orchestrating...
simulator: Python Simulator.py implementing the 7-phase drying cycle engine, MC decay model, and intelligent control loop. Listens to wood.kiln.control for automatic Ts and damper adjustments. Adjustments active only during active drying phases (not during Heating or Equalization).
fastapi: Python FastAPI backend exposing REST endpoints for dashboard interaction, cycle control, scenario management, and operator override of DT adjustments

scikit_learn: scikit-learn hierarchical Ward clustering for batch loading scenario D
z3: Z3 SMT solver for formal verification of MILP batch solutions. Runs automatically in background after MILP computation. Verifies P1 (minimum batch count), P2 (optimal MC spread), and P3 (optimal duration spread) by refutation. Results logged to docker logs.
rabbit_mq: RabbitMQ AMQP broker...
C12 DT constellation runs on the host machine. Communication uses AMQP (RabbitMQ) for sensor streaming, cycle start, and intelligent control; HTTP REST (FastAPI) for dashboard and scenario management.
C13 Twinning process and DT evolution Engineered following a POC-first approach for the course LOG6310E. 6 milestones: M1 simulator + RabbitMQ + Docker, M2 batch loading optimization (Greedy than MILP single-phase + Ward + Z3 formal verification), M3 Godot 3D visualization, M4 MC estimation + FastAPI dashboard, M5 deviation detection and recommendations, M6 intelligent control layer (Ts + dampers, active phases only). M1-M5: Digital Shadow. M6: full Digital Twin.
C14 Fidelity and validity considerations Models based on established wood science literature. Recipes R40/R44 and deviation rules derived from La Doré industrial practice. MC curves qualitatively consistent with industrial observations. K_DRYING not experimentally calibrated. Damper effect modeled as Th increase proportional to aperture (1.5C at 100%). No CFD airflow modeling. POC not formally validated against La Doré experimental data.
C15 DT technical connection AMQP via RabbitMQ (port 5672): sensor streaming, cycle start, and intelligent control commands. HTTP REST via FastAPI (port 8080): dashboard interaction and cycle control. All communication on localhost and Docker internal network. No physical kiln connection, PT emulated by simulator.
C16 DT hosting/deployment Deployed locally via Docker Compose.
C17 Insights and decision making batch_scenario: 4 batch loading scenarios computed by MILP and Ward clustering. Formally verified by Z3 for optimality on P1 (batch count), P2 (MC spread), P3 (duration spread).
formal_verification_result: Z3 formal verification results logged after MILP computation. Proves by refutation that no valid assignment exists with fewer batches (P1 proved optimal across all scenarios). Identifies residual suboptimality in MC spread (P2) for large groups under strict tolerance — a known NP-hard scalability limitation.
control_action: Automatic DT control actions published to wood.kiln.control every 10 real seconds minimum: Ts setpoint adjustment (+-2C, max cumulative +-6C) for SLOW_DRYING/FAST_DRYING/EXTEND_CYCLE/CHECK_OVERDRYING; damper aperture adjustment (25% steps, 20% minimum during active drying) for DTAB HIGH/LOW. Actions restricted to active drying phases only — not during Heating or Equalization. Operator can reset all adjustments via dashboard override button.
deviation_warning: Deviation status: ON_TARGET, SLOW_DRYING, FAST_DRYING, EXTEND_CYCLE, CHECK_OVERDRYING; DTAB alerts HIGH (>25C) / LOW (<5C) / NORMAL displayed on dashboard with color coding
mc_estimation_result: Real MC (exponential decay) and planned MC (linear reference) updated every 2 real seconds and displayed in dashboard charts and Godot UI
monitoring_dashboard: Web dashboard at port 8080 showing MC curves, temperature curves, phase progress, DTAB values, deviation status, damper aperture bar, Ts auto-adjustment indicator, and DT control action messages
post_drying_report: HTML report generated at cycle completion summarizing start/end time, final MC, duration deviation, Ts adjustment applied by DT, final damper opening, and recommendations
start_command: Operator-triggered kiln start via Godot UI after scenario validation; sends drying_group, width, initial_mc, total_pmp to wood.kiln.start
stop_command: Operator-triggered stop via dashboard Stop Kiln button; publishes stopped status to wood.kiln.sensor; resets Godot UI to door-open state
real_time_viz: 3D visualization of kiln state: steam particle intensity proportional to damper aperture percentage, fan rotation and direction reversal, coil light energy by Ts, door open/close animation
C18 Horizontal integration No horizontal integration with external systems. Standalone DT with synthetically generated inventory (200 lots with clustered MC distribution). No connection to external systems or other DTs.
C19 Data ownership and privacy Synthetic inventory data: no personal or sensitive information. Drying recipes and rules provided by La Doré industrial partner. MC model parameters informed by proprietary industrial data used within academic context only. No external data transmission.
C20 Standardization Communication via AMQP (RabbitMQ) and JSON payloads. Recipes and parameters follow La Doré industrial norms. Formal verification via Z3 SMT solver (SMT-LIB standard). Aligned with ISO 23247
C21 Security and safety considerations No physical safety risk: DT controls simulation only. Automatic control actions bounded: Ts adjustment capped at +-6C cumulative, dampers never below 20% during active drying. Local deployment with no external network exposure.
